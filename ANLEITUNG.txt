Wichtig:
Die Testversion speichert lokal. Für echte gemeinsame Daten aller Fahrer muss später

UPLOAD:
1. ZIP entpacken.
2. Die 5 Dateien in GitHub hochladen und vorhandene ersetzen.
3. Commit changes.
4. Seite neu laden. Der Service Worker dieser Version löscht alte Caches automatisch.

LIVE-SYNCHRONISIERUNG EINRICHTEN
1. Kostenloses Supabase-Projekt erstellen.
2. Datei SUPABASE_EINRICHTUNG.sql im Supabase SQL Editor ausführen.
3. Projekt-URL und öffentlichen anon key in der App unter Einstellungen eintragen.
4. Auf Verbinden tippen.

HINWEIS
Die Fotos werden in dieser Testversion als Bilddaten in der Datenbank gespeichert.
Für einen späteren Produktivbetrieb sollte Supabase Storage verwendet werden.
